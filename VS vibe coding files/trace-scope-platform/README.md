# Trace Scope Platform

Trace Scope Platform 是一个双核心的空间叙事平台，用于把“多图序列 360 度查看”和“基于地图的轨迹可视化”组织到同一个网站中。

## 本文件用途
本文件是给后续能力较弱模型的接手说明书，不是产品宣传文案。

建议阅读顺序：
1. 先读本文件
2. 再读 `docs/specs` 下的设计文档
3. 再读 `docs/plans` 下的实施计划
4. 最后再开始改代码

## 当前项目状态
- 已完成：目录结构、设计文档、实施计划、前后台页面骨架、共享类型、示例数据、基础路由
- 已完成：依赖安装、生产构建验证、Mapbox 容器接入、后台壳层补强
- 未完成：真实上传、真实排序、真实表单校验、真实持久化、鉴权
- 当前代码性质：可继续扩展的前端骨架，不是完整可交付产品

## 项目定位
- 前台站点：展示项目、轨迹、地点、360 图片序列和图集
- 后台站点：管理项目、地点、媒体组、图片和轨迹
- 视觉方向：空间档案馆、地理叙事、高对比度展陈式界面

## 核心数据链路
`Project -> Location -> MediaSet / Route`

这是强约束，不允许随意改变。

含义：
- `Project` 是顶层叙事单元
- `Location` 必须属于一个 `Project`
- `MediaSet` 必须属于一个 `Project`
- `Route` 必须属于一个 `Project`
- `MediaImage` 只能挂在 `MediaSet` 下，不能直接挂到页面

## 后续模型必须遵守的规则
- 不要把 `spin360` 和 `gallery` 合并成一个查看器组件
- 不要把 Mapbox 逻辑直接写进页面路由组件
- 保持路由页面轻量，并把业务类型集中管理
- 不要擅自改动核心实体字段名
- 不要擅自新增新的顶层核心实体
- 不要为了追求炫技破坏既定结构

## 目录说明
- `docs/specs`：设计规格
- `docs/plans`：实施计划
- `.planning/bootstrap`：启动阶段过程记录

## 关键代码边界
- 路由页面：只负责页面组合
- 共享类型：统一放在 `apps/web/src/types/domain.ts`
- 示例数据：统一放在 `apps/web/src/services/api/mock-data.ts`
- 地图模块：集中在 `apps/web/src/components/map`
- 媒体模块：集中在 `apps/web/src/components/media`
- 后台模块：集中在 `apps/web/src/app/routes/admin` 和 `apps/web/src/components/admin`

## 地图接入说明
- 真实地图依赖环境变量 `VITE_MAPBOX_ACCESS_TOKEN`
- 未配置 Token 时，页面仍会保留地图容器和交互边界，但不会显示真实底图
- 配置 Token 后重新运行 `npm run build` 或 `npm run dev` 即可验证地图加载
- 当前地图已支持点位标记、轨迹线渲染和地点选中同步的基础结构
